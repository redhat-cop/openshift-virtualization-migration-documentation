To access content stored on GitHub, your GitHub account must have credentials associated to your account. Two options are available:

. xref:github-credentials[GitHub Credentials]
. xref:bitwarden[Bitwarden]
. xref:ssh-key-generation[SSH Key Generation]

The following sections will describe how to configure your GitHub account with these credential types.

Login to link:https://github.com[GitHub] and navigate to the _SSH and GPG keys_ section of your Settings by clicking link:https://github.com/settings/keys[here].

Click the *New SSH key* button to begin the configuration process.

Paste the contents of the SSH Public Key previously copied in the _Key_ text field. Include an optional _Title_ to distinguish the key from other keys, and select the desired _Key type_ (The default option _Authentication Key_ is satisfactory).

Click the *Add SSH key* button to add the SSH Public Key to your account. You may be prompted to confirm your GitHub password.

To verify that you can successfully communicate with GitHub using the SSH key configured in your account, execute the following command.

[source,shell]
----
ssh -T git@github.com
----

If this is the first time you are connecting to GitHub, you will receive a warning indicating that you need to verify the authenticity of the host. Type `yes` and hit *Enter*.

The following message will be displayed indicating a successful connection to the GitHub server.

[source,text]
----
Hi jdoe! You've successfully authenticated, but GitHub does not provide shell access.
----

=== Personal Access Token

A GitHub link:https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/managing-your-personal-access-tokens[Personal Access Token (PAT)] can be used instead of SSH keys to authenticate to GitHub.

Login to link:https://github.com[GitHub] and navigate to the _Developer settings_ -> _Personal access tokens_ section of your Settings by clicking link:https://github.com/settings/tokens[here].

Click the *Generate new token* button to begin the configuration process.

Enter a _Note_ for the Personal Access Token (such as `openshift-virtualization-migration`) and optionally select an _Expiration_ date.

Under the _Select scopes_ section, select *repo* to grant the required repository access.

Click *Generate token* to create the token.

A confirmation page will be displayed and includes the value of the generated token. Be sure to copy the value as it cannot be accessed after the page is closed.

GitHub repositories can now be accessed using the _HTTPS_ protocol.

== Bitwarden

link:https://bitwarden.com[Bitwarden] is used to store sensitive resources for the Ansible for OpenShift Virtualization Migration Project and are located in a Bitwarden link:https://bitwarden.com/help/about-collections[Collection]. To gain access to the collection, you must have a Bitwarden account associated with your @redhat.com Email Address.

Use the following steps to complete the setup process.

. link:https://bitwarden.com/go/start-free[Create a Bitwarden account] using your Red Hat email address
. Join the `migration-factory` Google Group. The easiest way to join the Google Group is to onboard to the Ansible for OpenShift Virtualization Migration using link:https://docs.google.com/document/d/1zyx4vl6evt15jgHBcpaGOfcjcU6CSqUMLqQVBSGmdoE[this] form.
. Download one of the supported applications or use the Bitwarden website to access the resources that are available to you.

Once the membership to the _migration-factory_ Google Group has been confirmed, it may take some time until the Bitwarden Collection called _Migration Factory_ is available.

When the synchronization has completed, the _Migration Factory_ Collection should be present under the _Collection_ section of the Bitwarden Vault.

== SSH Key Generation

An SSH Key can be used to facilitate the communication between a local machine or the Ansible for OpenShift Virtualization Migration to external tools. The `ssh-keygen` command is used to create an SSH keypair for this purpose.

Create a new SSH keypair of type `rsa`. An example of the steps for generating the keypair is shown below. Be sure to update the email address associated with the `-C` parameter with your actual email address:

[source,shell]
----
